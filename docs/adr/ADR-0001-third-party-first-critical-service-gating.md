# ADR-0001: 第三方服务优先与关键服务阻断策略

日期: 2026-05-28

## 状态

Accepted

## 背景

Paper Audit / Veritas 审查的目标文献被定义为公开发表文献，不以隐私保护作为首要约束。项目目的不是尽量本地运行，而是通过第三方专业服务提升 OCR、语义审查、参考文献核验、图像语义分析和图像 AI 检测的覆盖率与质量。

当前实现中，本地轻量能力、第三方调用、缓存和降级报告混在一起。旧分析中曾建议隐私优先和第三方 opt-in，但这与本轮 grill 确认的产品方向不一致。

## 决策

默认策略为第三方服务增强优先。

关键能力失败时，不生成完整审查报告，而是生成失败诊断报告:

- 文本提取能力: MinerU 是绑定服务商。
- 文本语义审查能力: 按能力定义，可由 OpenAI-compatible LLM 等 Adapter 实现。
- 参考文献在线核验能力: 按能力定义，可由 Crossref/OpenAlex/PubMed 或其他 scholarly lookup Adapter 实现。
- 图像语义分析能力: 按能力定义，可由任意多模态 LLM Adapter 实现，不绑定 GLM。
- 图像 AI 概率检测能力: imagedetector 是绑定服务商。

阻断规则:

- MinerU preflight 或正式提取失败，阻断完整报告。
- 文本 LLM 任一分块最终失败，阻断完整报告。
- 如果识别到参考文献，默认必须尝试覆盖全部可解析参考文献；在线核验能力整体失败时生成 failed 产物。
- 如果输入中存在可检测图片，默认必须尝试覆盖全部可检测图片的图像语义分析和 imagedetector；对应能力整体失败时生成 failed 产物。
- 用户主动设置参考文献或图片覆盖上限、或主动关闭相关能力时，不能生成完整报告，只能生成 limited 产物。
- 没有参考文献或没有可检测图片时，对应能力不阻断。

关键服务失败必须写入正式产物目录，生成 Markdown + JSON，必要时生成 HTML。失败诊断报告必须列出失败服务、错误类型、配置/网络建议、已完成阶段、缓存状态和重试命令。

## 后果

完整报告比旧实现更严格，不再把外部服务失败默认为“部分报告成功”。

CLI、报告命名和测试需要区分三种产物:

- `audit_report.audit.*`: 完整报告。
- `audit_report.limited.*`: 用户显式限制范围后的范围受限报告。
- `audit_report.failed.*`: 关键服务失败后的诊断报告。

本地轻量能力仍可用于输入清单、诊断和快速检查，但不能替代关键第三方服务生成完整报告。
