# ADR-0002: 删除本地 LLM/OCR 正式审查路径

日期: 2026-05-28

## 状态

Accepted

## 背景

当前 README 和实现中存在本地 Ollama、本地 PDF 文本提取和 MinerU 失败后降级提取等路径。这些路径消耗本地资源，且不能满足项目确认的第三方服务增强优先目标。

用户明确确认: 完全删除本地 LLM 服务和本地 OCR 服务的正式审查路径。本地能力只保留轻量诊断边界。

## 决策

删除或废弃用户可见的本地正式审查入口:

- README 删除 Ollama/本地 LLM 作为支持路径的示例。
- `mykey.py` 自动兼容不再作为配置策略，见 ADR-0003。
- MinerU 失败后不自动降级成本地 OCR 或本地 PDF 正式审查。
- `--no-mineru`、`--no-reference-online`、`--no-image-semantic`、`--no-image-detector` 等关闭关键能力的参数，在正式审查模式下应删除或废弃。
- 如保留关闭参数，只能进入诊断/调试或范围受限模式，不能生成完整审查报告。

保留的本地轻量能力:

- 文件发现和输入清单。
- 轻量 PDF 文本直读用于诊断，不作为正式结论依据。
- 数字/统计快速检查。
- 缓存、run workspace、报告渲染。

## 后果

默认运行更依赖配置和网络，因此必须有 preflight。

用户若缺少关键服务配置，应收到失败诊断报告，而不是低质量本地兜底报告。

旧 CLI 和 README 需要迁移，避免继续承诺本地替代能力。
